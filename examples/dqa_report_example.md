### [P0 - 阻斷] 登入 API 會把密碼明文回傳
- **發生時間**: `2026-07-15 15:35:12`
- **現象描述**: 當輸入錯誤的密碼時，後端 API (`POST /api/login`) 的錯誤回傳 Payload 中，竟然包含了使用者剛剛輸入的明文密碼 `{"error": "Invalid password", "input": "123456"}`。
- **TEAM-U 維度**: T (可測性/異常流程) & Architecture (資安防線)。
- **RCA 初步分析**: RD 為了方便 Debug 將 req.body 直接丟進 Error Response 中，違反了 DQA 三不原則與資安防禦規定。
- **處理建議**: 必須立刻拿掉，此為 P0 等級的資安漏洞，修復前拒絕放行。

### [P1 - 嚴重] 首頁登入按鈕在 Safari 上無法點擊
- **發生時間**: `2026-07-15 15:42:05`
- **現象描述**: 在 macOS Safari 瀏覽器下，登入按鈕被背景圖層遮擋，導致無法觸發 `onClick` 事件。
- **TEAM-U 維度**: U (UI 可行性與跨瀏覽器相容)。
- **處理建議**: 請前端工程師調整 `z-index` 或檢查 flex 佈局。必須在交件前執行跨瀏覽器測試。

### [P3 - 次要] 錯誤提示的字體顏色太淺
- **發生時間**: `2026-07-15 15:50:30`
- **現象描述**: 帳號格式錯誤的紅色提示文字 (`#FFCCCC`) 在白色背景上對比度太低，違反無障礙設計 (a11y)。
- **處理建議**: 請調整色碼為深紅色 (`#CC0000`)。
